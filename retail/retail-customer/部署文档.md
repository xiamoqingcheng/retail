# 智能零售系统 - 微信小程序部署文档

## 一、项目概述

本项目是一个基于微信小程序的智能零售系统，支持扫码识别商品、购物车、订单结算等功能。

### 技术栈
- **前端**：微信小程序（WXML + WXSS + JavaScript）
- **后端**：Spring Boot + MyBatis
- **接口协议**：RESTful API
- **认证方式**：JWT Bearer Token

## 二、开发环境部署

### 2.1 前端部署（微信开发者工具）

#### 步骤 1：下载微信开发者工具
- 官网：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
- 下载对应系统的稳定版安装包
- 安装并登录微信小程序账号

#### 步骤 2：导入项目
1. 打开微信开发者工具
2. 点击「导入项目」
3. 选择项目目录：`c:\Users\ElNum\Desktop\software_system`
4. AppID：填入你的小程序 AppID（或使用测试号）
5. 点击「确认」

#### 步骤 3：配置编译模式
1. 在左侧菜单选择「详情」
2. 勾选「不校验合法域名」**（开发阶段）**
3. 编译模式选择「普通编译」

#### 步骤 4：编译运行
1. 点击「编译」按钮
2. 在模拟器或真机预览中测试

### 2.2 后端部署（本地开发）

#### 步骤 1：启动后端服务
```bash
cd backend
./mvnw spring-boot:run
# 或使用 IDE 启动 SpringBootApplication
```

#### 步骤 2：配置 ngrok 内网穿透（真机调试）

由于微信小程序真机调试需要 HTTPS 接口，本地开发需要使用 ngrok：

1. **下载 ngrok**
   ```bash
   # Windows
   # 下载 https://ngrok.com/download 并解压

   # 或使用包管理器
   scoop install ngrok
   choco install ngrok
   ```

2. **配置 Authtoken**
   ```bash
   ngrok config add-authtoken <你的authtoken>
   ```

3. **启动 ngrok**
   ```bash
   ngrok http 8080
   ```

4. **获取公网地址**
   ```
   Session Status                online
   Forwarding                    https://xxxxx.ngrok-free.app -> http://localhost:8080
   ```

5. **更新前端 API 地址**
   修改 `utils/request.js` 中的 `BASE_URL`：
   ```javascript
   const BASE_URL = 'https://xxxxx.ngrok-free.app/api';
   ```

6. **添加 ngrok 请求头**
   在 `utils/request.js` 的请求头中添加：
   ```javascript
   header: {
     'Content-Type': 'application/json',
     'Authorization': token ? 'Bearer ' + token : '',
     'ngrok-skip-browser-warning': '69420',
     'User-Agent': 'Mozilla/5.0',
     ...options.header
   }
   ```

## 三、生产环境部署

### 3.1 后端部署到云服务器

#### 服务器要求
- CPU：2核及以上
- 内存：4GB 及以上
- 系统：CentOS 7+ / Ubuntu 18+ / Windows Server
- 数据库：MySQL 5.7+

#### 部署步骤

1. **安装 Java 环境**
   ```bash
   # CentOS/Ubuntu
   sudo apt update
   sudo apt install openjdk-17-jdk

   # 验证
   java -version
   ```

2. **安装 MySQL**
   ```bash
   # CentOS
   sudo yum install mysql-server
   sudo systemctl start mysqld
   sudo systemctl enable mysqld

   # Ubuntu
   sudo apt install mysql-server
   sudo systemctl start mysql
   ```

3. **导入数据库**
   ```bash
   mysql -u root -p < database.sql
   ```

4. **上传后端 jar 包**
   ```bash
   scp app.jar user@server:/opt/retail-system/
   ```

5. **配置环境变量**
   ```bash
   export SPRING_PROFILES_ACTIVE=prod
   export DB_HOST=localhost
   export DB_PORT=3306
   export DB_NAME=retail_db
   export DB_USER=root
   export DB_PASSWORD=your_password
   ```

6. **启动服务**
   ```bash
   # 后台运行
   nohup java -jar app.jar --server.port=8080 > app.log 2>&1 &

   # 或使用 systemd
   sudo vim /etc/systemd/system/retail.service
   ```

   ```ini
   [Unit]
   Description=Retail Smart System
   After=network.target mysql.service

   [Service]
   Type=simple
   User=root
   ExecStart=/usr/bin/java -jar /opt/retail-system/app.jar
   Restart=always

   [Install]
   WantedBy=multi-user.target
   ```

   ```bash
   sudo systemctl daemon-reload
   sudo systemctl start retail
   sudo systemctl enable retail
   ```

7. **配置 SSL 证书**
   - 使用 Nginx 反向代理 + Let's Encrypt 证书
   - 或使用云服务商的负载均衡器配置 HTTPS

8. **配置 Nginx**
   ```nginx
   server {
       listen 443 ssl;
       server_name api.yourdomain.com;

       ssl_certificate /path/to/cert.pem;
       ssl_certificate_key /path/to/key.pem;

       location /api/ {
           proxy_pass http://127.0.0.1:8080/api/;
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
       }
   }
   ```

### 3.2 前端部署（生产环境）

#### 步骤 1：配置生产环境 API

修改 `utils/request.js`：
```javascript
// 生产环境
const BASE_URL = 'https://api.yourdomain.com/api';
```

#### 步骤 2：配置合法域名

1. 登录微信公众平台：https://mp.weixin.qq.com
2. 进入「开发」→「开发管理」→「开发设置」
3. 在「服务器域名」中添加：
   - request 合法域名：`https://api.yourdomain.com`
   - uploadFile 合法域名：`https://api.yourdomain.com`
   - downloadFile 合法域名：`https://api.yourdomain.com`

#### 步骤 3：上传代码

1. 在微信开发者工具中点击「上传」
2. 填写版本号和备注
3. 登录微信公众平台，在「版本管理」中提交审核

#### 步骤 4：发布小程序

1. 在微信公众平台「版本管理」中查看审核状态
2. 审核通过后，点击「发布」
3. 小程序将会在 1-24 小时内全量发布

## 四、常见问题

### Q1: 真机调试提示域名不合法
**解决**：
1. 开发阶段：勾选「详情」→「本地设置」→「不校验合法域名」
2. 生产阶段：在微信公众平台配置服务器域名

### Q2: ngrok 断开连接
**原因**：免费版 ngrok 会定期断开
**解决**：
1. 使用付费版 ngrok
2. 或使用国内内网穿透服务（natapp、樱花内网穿透等）
3. 或部署到云服务器

### Q3: 请求提示 JSON 格式错误
**解决**：检查后端接口文档，确认请求体格式（可能是直接传数组还是对象包裹）

### Q4: 图片显示不出来
**解决**：
1. 检查图片 URL 是否正确
2. 确保图片服务器支持 HTTPS
3. 在微信公众平台配置 downloadFile 合法域名
